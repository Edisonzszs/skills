# 路由规则

先判断当前请求属于哪个阶段。

## 走 Discovery

满足以下任一情况，先做 Discovery：
- 业务场景描述模糊
- 还没确认是否真的需要 AI
- 还没确认是否真的需要 Agent
- 试点范围不清楚
- 部署边界不清楚
- 合规边界不清楚
- 当前只是需求分析、可行性评估或方案选择阶段

## 走文档产出

仅当以下内容多数已经明确时，才进入 Product Brief / PRD / Agent Spec：
- 场景分类
- AI 必要性
- Agent 必要性
- 试点范围
- 部署与合规边界
- 人工审批点
- 风险与待确认事项

## 默认规则

判断不清时，先走 Discovery。
